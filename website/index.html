Features

Tunnels that manage themselves

Built for anyone who reaches databases, dashboards, and internal services over SSH. Fast, keyboard-friendly, security-first, and fully offline.

On-Demand Connect

The SSH session stays closed until something actually touches the local port. First byte in, Port Hippo dials the connection — the listener itself is always bound and ready.

Idle Teardown

When the last connection closes and the port goes quiet, the SSH session is torn down automatically — no lingering sessions or wasted sockets. The listener stays bound, ready to reconnect.

Jump-Host Chains

Reach hosts you can't hit directly. Chain any number of bastions and jump hosts; Port Hippo hops through them in order and forwards to the final destination.

Pause & Resume

Pause a tunnel to drop its live SSH session without deleting the definition. Resume any time — the port stays bound and reconnects on the next request.

Live Monitoring

Watch per-tunnel upload and download rates, byte totals, active connection counts, uptime, and last-active time update in real time in the Monitoring view.

Password or Key Auth

Authenticate each hop with a password or an SSH private key and passphrase. Secrets are encrypted at rest, and the UI never sees a decrypted credential.

Host-Key Verification

Every SSH host is checked against known_hosts. First contact prompts for explicit trust — Port Hippo never silently accepts an unknown key.

Local-Only by Default

Listeners bind to 127.0.0.1 out of the box — reachable only from your own machine. Exposing a tunnel to your LAN is a deliberate opt-in: you enter an explicit address:port, never the default.

Two Views, One Page

Define tunnels — local port, destination, SSH server, jump-host chain, and auth — in the Definition view; watch them run in Monitoring. Flip between them or show both side-by-side.

Connection Ref-Counting

Many clients can share one tunnel. Port Hippo ref-counts every live connection and only tears the SSH session down once the very last one has gone away.

No Tracking

No accounts, no telemetry, no analytics — nothing about you ever leaves your machine. The only outbound call is a background check to GitHub for updates. Its SSH transport (ssh2) is owned end-to-end: Port Hippo binds the local socket and relays the bytes itself.

Signed & Cross-Platform

Native builds for macOS, Windows, and Linux on both arm64 and x64 — signed and notarized where the platform supports it, with non-forced in-app auto-update.
